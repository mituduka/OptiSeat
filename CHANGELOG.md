# Changelog

本プロジェクトの主な変更点を記録します。
フォーマットは [Keep a Changelog](https://keepachangelog.com/ja/1.1.0/) に、

## [Unreleased]

### Added
- 解候補間の距離制約: 候補どうしで一定人数以上（既定: 児童・生徒の半数）の座席が変わることを保証（`SOLVER_MIN_DIFF_RATIO`）。数人の入れ替えだけの類似解や重複解が候補に並ばなくなり、十分に異なる解が尽きた場合は件数を水増しせずに返す

### Changed
- `SOLVER_RAND_FREQ` の既定値を `1.0` から `0.1` に変更。過剰なランダム化で探索誘導が無効化され解の品質が劣化していたため（同一時間内の最優先レベル違反が約半減）。解の多様性は距離制約側で保証する方式に変更（0.1 は品質劣化なしで実行間の第1候補の多様性も保てる実測値）
- 前後配慮の探索誘導（`#heuristic`）の modifier を `true` から `sign` に変更。`true` は決定順を固定するため配慮タグの児童・生徒が実行間・候補間で毎回同じ席に張り付いていた。`sign` は誘導効果を保ったままエリア内での席が分散する

### Fixed
- マルチショットのシード変更が並列探索スレッドの1本にしか適用されていなかった問題を修正（全スレッドに設定）
- 前後配慮エリアを0行にしたときの解釈を全レイヤで統一。従来は後側エリア0行が「最終行」に化けて back_preferred が最終行へ誘導され、前側エリア0行はソルバが無視する一方でスコア・バッジは全員違反として表示していた。0行（空リスト）は「エリアなし＝配慮タグは対象外（違反0件・バッジ非表示）」とし、配慮タグ保持者がいるのにエリアが0行の場合は実行画面に警告を表示する（`back_rows` の省略/null は従来どおり最終行を自動設定）

## [1.0.0] - 2026-06-17

初回公開リリース。

### Added
- ASP（clingo + clorm）による席替えの制約充足・最適化と複数解の提示
- ハード制約: 座席固定・8方向隣接禁止・同班禁止・座席性別制約・隣接配置（relative_fixed）・班分散グループ（leader_group）
- ソフト制約: 前後配慮・班内男女バランス・孤独感緩和・前回席/班/横隣の差分
- 名簿 → 条件 → 実行 → 調整 → 結果 の 5 ステップ UI（ステップナビで進行状況と完了状態を常時表示、各画面に「次へ」導線）
- デジタル庁デザインシステムを参考にしたデザイントークン（プライマリ: 青竹ティール、セマンティックカラーは境界 3:1・テキスト 4.5:1 を保証）
- UI 全体で色の意味づけを統一（プライマリ=リンク・選択・設定あり、青/ピンク=性別、アンバー=配慮・制約の注釈、赤=違反・削除、紫=班分散グループ、グレー=空席・無効）
- 座席セルの 2 段固定レイアウト（氏名ゾーン＋注釈ゾーン）により、バッジ・ラベルの有無によらず全セルで氏名位置が揃う（画面・PDF 共通）
- 配置調整のドラッグ＆ドロップ（@dnd-kit）とモバイル向けタップ交換モード（キーボード操作にも対応）
- 座席グリッドの自動縮小と横スクロールヒント（モバイルでも座席表全体を確認可能）
- アクセシビリティ対応: フォーカス対応ツールチップ、`role="alert"` / `role="status"` による通知、モーダルのフォーカストラップ、サイト全体で統一したフォーカスリング、本文 14px 以上・行間 160%
- 座席表のクライアントサイド・ベクター PDF 出力（端末非依存、黒板バー・氏名帯・班色・班分散ラベルなど画面と同一の配色）
- 座席表タイトルの設定（結果表示画面で入力。PDF 上部とダウンロードファイル名に反映され、エクスポート JSON にも記録）
- FastAPI サーバーモードによる席替え計算 API
- 制約矛盾の事前チェック（validate API）と違反箇所のリアルタイム表示
- サンプルデータ挿入機能、JSON エクスポート / インポート（取り込み時に構造検証）
- README 用スクリーンショットの自動撮影スクリプト（Playwright、`npm run screenshots`）
- 法務ドキュメント一式（利用規約・プライバシーポリシー・ライセンス情報・第三者ソフトウェア表示）と `/legal` ページ
- 環境変数サンプル（`.env.example` / `backend/.env.example` / `frontend/.env.example`）

### Security
- プライバシー設計: 児童・生徒の氏名はブラウザの `localStorage` にのみ保存し、API には匿名 ID のみ送信
- CORS は許可オリジンを限定し、資格情報（Cookie 等）を使用しない構成
